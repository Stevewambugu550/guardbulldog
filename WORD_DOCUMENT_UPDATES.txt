SECTION 2: DATABASE ENHANCEMENTS
================================================================================

2.1 NEW DATABASE TABLES (8 Tables Created)

Table 1: guest_reports
   Purpose: Store anonymous phishing report submissions
   Key Fields:
   - id (Primary Key)
   - tracking_token (Unique identifier for guests)
   - email (Optional contact email)
   - subject (Email subject line)
   - description (Detailed report)
   - status (Pending, Investigating, Resolved)
   - ip_address (Submitter IP for tracking)
   - created_at, resolved_at

Table 2: ip_intelligence
   Purpose: Store IP address reputation and geolocation data
   Key Fields:
   - id (Primary Key)
   - ip_address (Unique)
   - country, city, region, isp
   - threat_level (Safe, Suspicious, Dangerous)
   - reputation_score (0-100)
   - is_blacklisted (Boolean)
   - report_count (Number of times reported)
   - first_seen, last_seen

Table 3: email_sources
   Purpose: Store detailed email header and routing information
   Key Fields:
   - id (Primary Key)
   - report_id (Link to reports table)
   - guest_report_id (Link to guest_reports)
   - sender_email, sender_ip, sender_domain
   - mail_server
   - email_headers (JSONB for flexible storage)
   - routing_info (JSONB)
   - spf_status, dkim_status, dmarc_status
   - email_received_date, email_sent_date

Table 4: phishing_detection_results
   Purpose: Store AI analysis results for each report
   Key Fields:
   - id (Primary Key)
   - report_id, guest_report_id
   - risk_score (0-100)
   - classification (Safe, Suspicious, Phishing, Malware)
   - confidence_level (0-100%)
   - indicators_found (JSONB array)
   - suspicious_patterns (JSONB)
   - url_analysis (JSONB)
   - ai_model_version
   - analyzed_at

Table 5: user_inquiries
   Purpose: Store user-submitted questions and support requests
   Key Fields:
   - id (Primary Key)
   - user_id (Authenticated users)
   - guest_email (For anonymous inquiries)
   - subject, message
   - priority (Low, Normal, High, Urgent)
   - status (New, In Progress, Resolved, Closed)
   - category (General, Technical, Report, Feedback)
   - assigned_to (Admin user ID)
   - created_at, first_response_at, resolved_at

Table 6: inquiry_responses
   Purpose: Store conversation thread for each inquiry
   Key Fields:
   - id (Primary Key)
   - inquiry_id
   - user_id (Responder)
   - message
   - is_admin_response (Boolean)
   - is_internal_note (Boolean - admin only)
   - attachments (JSONB)
   - created_at

Table 7: threat_indicators
   Purpose: Database of known malicious indicators
   Key Fields:
   - id (Primary Key)
   - indicator_type (URL, Domain, IP, Hash, Email)
   - indicator_value
   - threat_type (Phishing, Malware, Spam, Scam)
   - severity (Low, Medium, High, Critical)
   - source (Manual, Automated, External Feed)
   - first_detected, last_detected
   - detection_count
   - is_active

2.2 ENHANCED EXISTING TABLES

users Table - New Fields Added:
   - user_type: Student, Faculty, Staff, Admin classification
   - department: Academic or administrative department
   - academic_year: For student users
   - job_title: For faculty and staff
   - vulnerability_score: Calculated risk score (0-100)
   - training_completed: Security awareness status
   - phishing_tests_passed: Number of simulations passed
   - phishing_tests_failed: Number of simulations failed
   - last_training_date: Most recent training completion

reports Table - New Fields Added:
   - email_received_date: When victim received the email
   - email_sent_date: Original send timestamp
   - first_viewed_at: Admin first review time
   - resolved_at: Resolution completion time
   - sender_ip: Extracted from email headers
   - risk_score: AI-calculated threat level (0-100)
   - auto_classification: AI threat category
   - confidence_level: AI confidence percentage
   - target_type: Student, Faculty, or Staff
   - attack_vector: Email, SMS, Voice, Social Media

2.3 PERFORMANCE OPTIMIZATIONS

Indexes Created: 25+ High-Performance Indexes

   - Full-text search indexes for content search

Automated Triggers: 6 Database Triggers

Analytical Views: 3 Database Views

SECTION 4: API ENHANCEMENTS
================================================================================

4.1 NEW API ENDPOINTS (12 New Endpoints)

Guest Access APIs:
   POST   /api/reports/guest-submit
          Submit phishing report without authentication
          Rate Limited: 5 requests per hour per IP

   GET    /api/reports/track/:token
          Track report status using tracking token
          Public access with token validation

IP Intelligence APIs:
   GET    /api/intelligence/ip/:address
          Get IP reputation and geolocation data
          Returns: country, city, threat level, score

   GET    /api/intelligence/domain/:domain
          Get domain reputation information
          Returns: age, reputation, threat status

   POST   /api/intelligence/analyze
          Comprehensive email intelligence analysis
          Returns: IP data, domain info, threat assessment

AI Detection APIs:
   POST   /api/ai/scan-email
          Analyze email content for phishing indicators
          Returns: risk score, classification, confidence

   GET    /api/ai/threat-level/:reportId
          Get AI threat assessment for specific report
          Returns: detailed analysis results

   GET    /api/ai/indicators/:reportId
          Get list of detected threat indicators
          Returns: URLs, keywords, patterns found

User Inquiry APIs:
   POST   /api/inquiries/submit
          Submit user inquiry or support request
          Supports both authenticated and guest users

   GET    /api/inquiries/user
          Get user's inquiry history
          Requires authentication

   GET    /api/inquiries/admin
          Get all inquiries for admin review
          Admin access only

   PUT    /api/inquiries/:id/respond
          Add response to inquiry thread
          Admin or original user

Enhanced Analytics APIs:
   GET    /api/analytics/by-user-type
          Reports grouped by user demographic
          Returns: student, faculty, staff breakdown

   GET    /api/analytics/ip-threats
          Geographic threat distribution
          Returns: country-based attack map data

   GET    /api/analytics/time-series
          Temporal trend analysis
          Returns: time-based attack patterns

   GET    /api/analytics/ai-performance
          AI detection accuracy metrics
          Returns: accuracy, false positives, speed
